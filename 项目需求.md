# 进程调度与内存管理模拟器 — 项目需求文档

## 项目简介

用纯 C 语言模拟操作系统的**进程调度**与**内存分配**核心机制，通过交互式命令行界面驱动。

---

## 四种链表的分工

| 链表类型 | 用在哪里 | 原因 |
|---|---|---|
| 单向链表 | FCFS 就绪队列 | 只需从头取、从尾插，单向足够 |
| 单向循环链表 | Round Robin 调度队列 | 天然循环，时间片到了直接转下一个 |
| 双向链表 | 内存块管理空闲链表 | 合并相邻空闲块时需要访问前驱节点 |
| 双向循环链表 | 优先级调度队列 | 同优先级进程循环轮转，支持前后插队 |

## 核心数据结构

**进程控制块 PCB：**

```c
typedef struct {
    int          pid;
    char         name[32];
    int          priority;        // 1(最高) ~ 10(最低)
    int          burst_time;      // 需要的 CPU 总时间 (ms)
    int          remaining_time;  // 剩余时间（RR 用）
    int          arrival_time;
    int          finish_time;
    int          waiting_time;
    int          turnaround_time;
    ProcessState state;           // NEW/READY/RUNNING/WAITING/TERMINATED
} Process;
```

**内存块描述符：**

```c
typedef struct {
    int start_addr;   // 起始地址 (KB)
    int size;         // 大小 (KB)
    int pid;          // -1 表示空闲
    int is_free;
} MemBlock;
```

## 功能模块需求

### 模块一：四种链表库

所有链表基于 `void *` 泛型实现，用函数指针支持比较和释放，接口风格统一。

**单向链表** 需实现：
- `insert_head` / `insert_tail` / `insert_at`
- `delete_head` / `delete_tail` / `delete_at`
- `find` / `traverse` / `reverse`

**单向循环链表** 需实现：
- `enqueue`（尾插）
- `get_current`（取 head 数据）
- `advance`（head 指针后移一步）
- `remove_current`（删除 head 节点）
- `traverse`

**双向链表** 需实现：
- `insert_head` / `insert_tail`
- `insert_before(node)` / `insert_after(node)`
- `delete_node(node)`（传节点指针，O(1) 删除）
- `traverse_forward` / `traverse_backward`

**双向循环链表** 需实现：
- `insert_after_current`
- `insert_sorted`（按优先级有序插入）
- `move_next` / `move_prev`
- `remove_current`
- `traverse`

---

### 模块二：进程调度器

#### ① FCFS—— 使用单向链表

- 按 `arrival_time` 将进程入队
- 逐个出队模拟运行
- 输出甘特图（文本形式）：

```
[P1:0~5][P2:5~12][P3:12~15]
```

- 计算并输出每个进程的等待时间、周转时间，最后输出平均值

#### ② Round Robin —— 使用单向循环链表

- 用户可配置时间片大小（默认 4ms）
- 每轮给 `current` 进程分配一个时间片，`remaining_time` 减少
- `remaining_time <= 0` 时调用 `remove_current`，否则调用 `advance`
- 输出每个时间片的执行情况及最终统计

#### ③ 优先级调度 —— 使用双向循环链表

- 通过 `insert_sorted` 按优先级有序插入
- 相同优先级之间用 `move_next` / `move_prev` 循环轮转
- 高优先级进程永远先执行
- 支持运行过程中动态插入新进程（插入后自动调整位置）

---

### 模块三：内存管理器 —— 使用双向链表

模拟一块 **512 KB** 的内存，用双向链表管理所有内存块（无论空闲还是占用）。

需实现：

- `first_fit_alloc(pid, size)`：首次适应，找第一个够大的空闲块
- `best_fit_alloc(pid, size)`：最佳适应，找最小的够用空闲块
- `free_mem(pid)`：释放指定进程的内存，并立即合并相邻空闲块
- `merge_free_blocks()`：遍历链表，将相邻空闲块合并为一个
- `visualize()`：文本可视化内存布局，例如：

```
[内存布局 512KB]
|0          |128        |256        |384        |512
|-- P1(64K)-|-- FREE ---|-P2(128K)--|-- FREE ---|
```

### 模块五：测试用例

**`tests/test_lists.c`** — 四种链表单元测试：
- 边界条件：空表操作、单节点、插入到头/尾/中间
- 正确性：插入 N 个元素后 `size` 是否正确，遍历顺序是否正确
- 内存安全：用 `valgrind --leak-check=full` 跑，无泄漏

**`tests/test_scheduler.c`** — 提供固定进程集，验证三种调度结果的正确性。

## Makefile 要求

支持以下目标：

```makefile
make           # 编译主程序 → bin/scheduler_sim
make test      # 编译并运行 tests/ 下的测试
make valgrind  # 用 valgrind 检测内存泄漏
make clean     # 清除所有编译产物
```

编译选项：`-Wall -Wextra -std=c99 -Iinclude`

1. 完成四种链表并全部测试通过
2. 定义 `types.h`，实现进程的创建与删除
3. 实现三种调度算法（从 FCFS 开始）
4. 实现内存管理器
5. 接入 `main.c` 菜单
6. 补全 Makefile 和 README
